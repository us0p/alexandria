## Clean Architecture
It's a suggestion of how to split your code into different components.
## Domain-Driven Design
Is an approach to developing systems where the underlying idea is to give you a clear mental map of the domain you're working with.

It's a set o concepts, tools and practices that you can use to explore the domain.
## Clean Architecture and Domain-Driven Design
Domain-Driven Design is often used to implement the core of domain inside the Clean Architecture.
When you follow Domain-Driven Design it makes sense to use a domain centric architectural pattern.
